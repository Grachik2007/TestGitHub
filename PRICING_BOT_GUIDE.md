# 💰 Pricing Bot Integration Guide

## Обзор

Телеграм бот теперь полностью интегрирован с системой управления ценами. Вы можете управлять всеми параметрами ценообразования прямо из бота в реальном времени!

## 🚀 Запуск

### 1. Запустить Pricing API

```bash
node pricing-api.js
```

Сервер запустится на порту 3001 (или `$PORT`):
```
💰 === PRICING API ЗАПУЩЕНА ===
🌐 http://localhost:3001/api/v1/pricing
```

### 2. Запустить Telegram бот

```bash
export TELEGRAM_BOT_TOKEN="your_token_here"
export PRICING_API_URL="http://localhost:3001"
python apps/telegram/main.py
```

**Переменные окружения:**
- `TELEGRAM_BOT_TOKEN` - токен от BotFather в Telegram
- `PRICING_API_URL` - адрес Pricing API (по умолчанию http://localhost:3001)
- `API_URL` - адрес основного API (по умолчанию http://localhost:8000)

## 📱 Команды бота

### /prices - Управление ценами

**Показать текущие параметры:**
```
/prices
или
/prices view
```

Выведет:
```
📊 ТЕКУЩИЕ ПАРАМЕТРЫ ЦЕН

💵 Фиксированные затраты:
  • Упаковка: 200₽
  • Сборка: 400₽
  • Себестоимость (по умолчанию): 1000₽

📈 Комиссии и расходы:
  • Комиссия эквайринга: 12.0%
  • Маркетинг: 5.0%
  • Налог УСН: 20.0%

🎯 Целевая маржа:
  • 30%
```

### Изменение параметров

**Упаковка (в рублях):**
```
/prices packaging 300
```
Ответ: `✅ Упаковка обновлена: 300₽`

**Сборка (в рублях):**
```
/prices assembly 150
```
Ответ: `✅ Сборка обновлена: 150₽`

**Комиссия эквайринга (в долях, где 1.0 = 100%):**
```
/prices commission 0.04
```
Ответ: `✅ Комиссия обновлена: 4.0%`

**Маркетинг (в долях):**
```
/prices marketing 0.06
```
Ответ: `✅ Маркетинг обновлен: 6.0%`

**Целевая маржа (в долях):**
```
/prices margin 0.35
```
Ответ: `✅ Маржа обновлена: 35%`

### Пересчет цен

После изменения параметров пересчитать все цены:
```
/prices recalc
```

Бот выведет:
```
⏳ Пересчитываю цены... Это может занять минуту...
[ждет завершения...]
✅ Цены успешно пересчитаны и feeds обновлены!
```

Это:
1. Запустит parser-test.js
2. Пересчитает все цены на основе новых параметров
3. Обновит feeds/products.yml и feeds/products.csv
4. Готов к синхронизации с GitHub Pages

## 🔐 API endpoints (для интеграции)

Если захочешь вызывать API напрямую (не через бота):

**GET текущей конфигурации:**
```bash
curl http://localhost:3001/api/v1/pricing
```

**Обновить параметры:**
```bash
curl -X POST http://localhost:3001/api/v1/pricing \
  -H "Content-Type: application/json" \
  -d '{
    "costs": {"packaging": 300},
    "margin": {"target": 0.35}
  }'
```

**Пересчитать цены:**
```bash
curl -X POST http://localhost:3001/api/v1/pricing/recalculate
```

**Примеры:**
```bash
curl http://localhost:3001/api/v1/pricing/examples
```

## 🛠️ Структура конфига

`config/pricing.json` - главный конфиг:

```json
{
  "costs": {
    "packaging": 200,      // упаковка в рублях
    "assembly": 400,       // сборка в рублях
    "defaultWholesaleCost": 1000
  },
  "commissions": {
    "acquiring": 0.12      // комиссия эквайринга (0.12 = 12%)
  },
  "expenses": {
    "marketing": 0.05,     // маркетинг (5%)
    "tax": 0.20            // налог УСН (20%)
  },
  "margin": {
    "target": 0.30         // целевая маржа (30%)
  }
}
```

## 💡 Формула ценообразования

```
fixedCosts = себестоимость + упаковка + сборка
variableRatio = комиссия + маркетинг + (налог × (1 - маржа))

цена = fixedCosts / (1 - variableRatio)
```

**Пример для товара с себестоимостью 1500₽:**
```
fixedCosts = 1500 + 200 + 400 = 2100
variableRatio = 0.12 + 0.05 + (0.20 × 0.70) = 0.31
цена = 2100 / 0.69 ≈ 3043₽
```

## 🚨 Troubleshooting

**❌ "Не смог получить конфигурацию цен"**
- Проверь, запущена ли Pricing API: `node pricing-api.js`
- Проверь переменную `PRICING_API_URL` в боте
- Убедись, что API слушает на нужном портуhome (по умолчанию 3001)

**❌ "Ошибка пересчета"**
- Проверь что установлены зависимости: `npm install`
- Убедись что `config/pricing.json` существует
- Посмотри логи: `tail -f pricing-api.js`

**❌ "Неизвестная команда"**
- Используй правильный синтаксис: `/prices packaging 300` (числа через пробел)
- Для комиссии/маркетинга/налога используй доли: `0.04` вместо `4`
